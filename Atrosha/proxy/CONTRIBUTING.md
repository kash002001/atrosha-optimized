# Contributing to Atrosha Kernel

First off, thanks for taking the time to contribute!

## Code of Conduct
This project and everyone participating in it is governed by the Atrosha Code of Conduct. By participating, you are expected to uphold this code.

## How Can I Contribute?
### Reporting Bugs
Before creating bug reports, please check the existing issues as you might find out that you don't need to create one. When you are creating a bug report, please include as many details as possible.

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please include a step-by-step description of the suggested enhancement.

### Pull Requests
* Fill in the required template.
* Do not include more than one fix/feature per pull request.
* Ensure code passes all CI checks.
* Follow Rust best practices and `cargo fmt`.

## Styleguides
### Git Commit Messages
* Use the present tense ("Add feature" not "Added feature")
* Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
* Limit the first line to 72 characters or less

### Rust Styleguide
* We use `rustfmt` to maintain a consistent style.
* Documentation comments are highly encouraged for public APIs.
